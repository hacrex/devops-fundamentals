# Phase 4: Observability

This phase covers monitoring, logging, and tracing for DevOps.

## Topics Covered

1. **Monitoring Basics** - Metrics, alerting, monitoring strategies
2. **Prometheus** - Metrics collection and storage
3. **Grafana** - Visualization and dashboards
4. **Logging with ELK** - Centralized log management
5. **Distributed Tracing** - Request tracing across services

## Learning Path
```
Monitoring Basics → Prometheus → Grafana → ELK Stack → Distributed Tracing
```

## Prerequisites
- Complete Phase 3 (Containers & Kubernetes)
- Basic understanding of microservices

## Topics

### [Topic 1: Monitoring Basics](./topic-01-monitoring-basics.md)
Learn the fundamentals of monitoring including the three pillars of observability, metric types, and alerting strategies.

### [Topic 2: Prometheus](./topic-02-prometheus.md)
Deep dive into Prometheus architecture, PromQL, service discovery, and alerting.

### [Topic 3: Grafana](./topic-03-grafana.md)
Create powerful dashboards and visualizations using Grafana with multiple data sources.

### [Topic 4: Logging with ELK](./topic-04-elk-logging.md)
Implement centralized logging using Elasticsearch, Logstash, and Kibana.

### [Topic 5: Distributed Tracing](./topic-05-distributed-tracing.md)
Track requests across microservices using Jaeger and OpenTelemetry.

## Hands-On Labs

1. Deploy Prometheus + Grafana stack
2. Create monitoring dashboards
3. Set up alerting rules
4. Configure centralized logging
5. Implement distributed tracing
