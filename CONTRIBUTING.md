# 🤝 Contributing to DevOps Fundamentals

First off, thank you for considering contributing! This project exists to help beginners learn DevOps, and every contribution makes it better.

---

## 🌟 Ways to Contribute

- 📝 Add or improve topic notes
- 🧪 Add new hands-on labs
- 🐛 Fix typos or incorrect information
- 📚 Add free learning resources
- 💡 Suggest new topics via Issues
- ⭐ Star the repo to help others find it

---

## 🚀 Getting Started

1. **Fork** the repository
2. **Clone** your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/devops-fundamentals.git
   cd devops-fundamentals
   ```
3. **Create a branch**:
   ```bash
   git checkout -b feature/add-ansible-notes
   ```
4. **Make your changes**
5. **Commit** with a clear message:
   ```bash
   git commit -m "Add Ansible playbook examples to phase-6"
   ```
6. **Push** and open a **Pull Request**:
   ```bash
   git push origin feature/add-ansible-notes
   ```

## 📋 Contribution Guidelines

### Content Standards
- Write in simple, beginner-friendly language
- Use practical examples over theory
- Include code snippets where possible
- Add a TL;DR or summary at the end of long docs
- Keep files under 500 lines — split if needed

### File Naming
- Use lowercase with hyphens: `my-topic-name.md`
- Follow the existing folder structure
- Number files that are part of a sequence: `01-intro.md`

### Markdown Style
- Use headers (`##`, `###`) to organize content
- Use code blocks with language hints (` ```bash `, ` ```yaml `)
- Use tables for comparisons
- Add emojis sparingly for visual scanning

### What We Don't Accept
- Paid course promotions
- Content requiring paid tools (prefer open source / free tier)
- Plagiarized content
- Off-topic content

## 🐛 Reporting Issues

Use GitHub Issues to report:
- Incorrect information
- Broken links
- Missing topics
- Suggestions for improvements

Please use the issue templates provided.

---

## 📜 Code of Conduct

- Be respectful and inclusive
- Welcome beginners — everyone starts somewhere
- No gatekeeping or elitism
- Help others learn — explain, don't dismiss

---

## ✅ PR Checklist

Before submitting a PR, make sure:
- [ ] Content is accurate and tested
- [ ] No typos or grammatical errors
- [ ] Follows the file naming convention
- [ ] No duplicate content
- [ ] Links are working
